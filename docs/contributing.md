# Contributing

## 1. Общие правила
- Все изменения проходят через PR.
- Обновления документации обязательны при изменении поведения.
- Важно соблюдать структуру `apps/`, `packages/`, `infra/`.

## 2. Ветки и коммиты
- Ветки: `feature/<scope>`, `fix/<scope>`, `docs/<scope>`.
- Коммиты: короткие, в повелительном наклонении.

## 3. Минимальный checklist
- Локально запускаются tests (если есть).
- Добавлены/обновлены документы в `docs/`.
- Нет новых линт-ошибок.

## 4. Архитектурные изменения
- Все ключевые решения фиксируются в `docs/architecture-decisions.md`.
