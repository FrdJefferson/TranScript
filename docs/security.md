# Security

## 1. Threat model (коротко)
- Несанкционированный доступ к приватным гайдам и экспортам.
- Утечки исходного видео/аудио и транскриптов.
- Злоупотребление API (перегрузка воркеров, обход лимитов).
- Утечка токенов шаринга.

## 2. Доступ и авторизация
- По умолчанию гайды приватные.
- Доступ владельцу, роли через `GuideAccess`.
- Публичные ссылки read-only с `ShareLink`.

## 3. Секреты и ключи
- Все ключи (ASR/LLM, storage) в переменных окружения.
- Ротация секретов и минимальные права (least privilege).

## 4. Транспорт и хранение
- TLS везде (API, storage).
- Шифрование at-rest для S3 и БД (при наличии).

## 5. PII и логирование
- Не логировать сырые тексты/транскрипты.
- Логи: id сущностей, статусы, traceId.

## 6. Политика удаления
- Удаление по запросу пользователя удаляет все связанные артефакты.
- Retention TTL применяются фоновыми джобами.
